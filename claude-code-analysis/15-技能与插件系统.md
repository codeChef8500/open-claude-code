## 1. 模块职责概述

Claude Code 通过两个扩展机制让用户自定义工作流：
- **技能（Skills）**：Markdown 定义的可重用工作流，以 Slash 命令形式触发
- **插件（Plugins）**：NPM 包或本地目录，可提供工具、命令和 hooks

## 2. 技能系统（Skills）

### 2.1 技能文件结构

技能是存放在 `.claude/commands/` 或 `~/.claude/commands/` 下的 Markdown 文件：

```markdown
<!-- .claude/commands/fix-pr.md -->
---
description: 分析 PR 评论并修复问题
allowed-tools: BashTool, FileEditTool, FileReadTool
argument-hint: <PR 编号>
activateWhen:
  filePattern: "*.ts"
---

你是一个 PR 修复专家。请分析 PR #$ARGUMENTS 的评论，理解每个
评论的意图，然后在代码中修复对应的问题。

修复完成后：
1. 运行测试确保没有引入新问题
2. 提交更改并推送到分支
```

### 2.2 Frontmatter 字段

```typescript
type SkillFrontmatter = {
  description: string           // 技能描述（显示在 /help 中）
  'allowed-tools'?: string[]    // 技能可以使用的工具列表（白名单）
  'argument-hint'?: string      // 参数提示（自动补全提示）
  model?: 'sonnet' | 'opus' | 'haiku'  // 指定使用的模型
  activateWhen?: {
    filePattern?: string        // 文件匹配模式（条件激活）
    envVar?: string             // 环境变量条件
  }
  subagentType?: string         // 指定子 Agent 类型
}
```

### 2.3 `loadSkillsDir.ts` — 技能加载（34KB）

```typescript
// 从目录加载所有技能，解析为 Command 对象
export async function getSkillDirCommands(cwd: string): Promise<Command[]> {
  const skillDirs = getSkillDirectories(cwd)  // 搜索 .claude/commands/
  
  const commands: Command[] = []
  
  for (const dir of skillDirs) {
    const files = await glob('**/*.md', { cwd: dir })
    
    for (const file of files) {
      const content = await readFile(join(dir, file), 'utf-8')
      const { frontmatter, body } = parseFrontmatter(content)
      
      // 将 Markdown 技能文件转换为 PromptCommand
      commands.push({
        type: 'prompt',
        name: fileNameToCommandName(file),  // fix-pr.md → fix-pr
        description: frontmatter.description,
        source: 'custom',
        async getPromptForCommand(args) {
          // $ARGUMENTS 占位符替换
          return body.replace(/\$ARGUMENTS/g, args)
        },
        // 工具白名单
        ...(frontmatter['allowed-tools'] && {
          allowedTools: frontmatter['allowed-tools'],
        }),
      })
    }
  }
  
  return commands
}
```

### 2.4 技能搜索（`services/skillSearch/`，EXPERIMENTAL_SKILL_SEARCH）

```typescript
// 当技能数量较多时，支持语义搜索发现相关技能
// 使用本地向量索引（不调用 API）
async function searchSkills(query: string): Promise<Skill[]> {
  const index = await loadSkillIndex()
  
  // TF-IDF 或简单关键词匹配
  return index
    .filter(skill => skill.keywords.some(k => query.includes(k)))
    .sort((a, b) => relevanceScore(b, query) - relevanceScore(a, query))
    .slice(0, 5)
}
```

// 当文件被编辑时，检查是否触发了新的技能目录发现
export async function discoverSkillDirsForPaths(paths: string[]): Promise<void> {
  for (const path of paths) {
    const skillDir = findSkillDirForPath(path)
    if (skillDir) {
      await addSkillDirectories([skillDir])
      
      // 通知 QueryEngine 有新技能可用
      appendSystemMessage(`发现新技能目录: ${skillDir}`)
    }
  }
}

// 条件激活：文件匹配 activateWhen.filePattern 时，激活对应技能
export async function activateConditionalSkillsForPaths(paths: string[]): Promise<void> {
  const skills = await loadAllSkills()
  
  for (const skill of skills) {
    if (skill.frontmatter.activateWhen?.filePattern) {
      const pattern = skill.frontmatter.activateWhen.filePattern
      if (paths.some(p => minimatch(p, pattern))) {
        await activateSkill(skill)  // 将技能描述注入系统提示
      }
    }
  }
}

## 4. 插件系统（Plugins）

插件是更强大的扩展机制，可以提供：
- 新工具（`contributes.tools`）
- 新命令（`contributes.commands`）
- Hooks（`contributes.hooks`）
- 自定义 UI 组件

### 4.1 插件包格式（`package.json`）

```json
{
  "name": "@company/claude-plugin-jira",
  "version": "1.0.0",
  "main": "index.js",
  "claudePlugin": {
    "version": "1",
    "contributes": {
      "tools": ["JiraTool"],
      "commands": ["jira-comment", "jira-status"],
      "hooks": {
        "preToolUse": "hooks/preToolUse.js",
        "postToolUse": "hooks/postToolUse.js"
      }
    }
  }
}
```

### 4.2 `utils/plugins/loadPluginCommands.ts` — 插件加载

```typescript
// 从插件目录加载所有插件命令
export async function getPluginCommands(): Promise<Command[]> {
  const pluginDirs = await getInstalledPluginDirs()
  
  const commands: Command[] = []
  
  for (const dir of pluginDirs) {
    const pkg = await loadPluginPackage(dir)
    
    // 验证插件格式
    if (!validatePlugin(pkg)) {
      logError(`插件 ${dir} 格式无效`)
      continue
    }
    
    // 加载插件命令
    const pluginModule = await import(join(dir, pkg.main))
    
    for (const commandName of pkg.claudePlugin.contributes.commands) {
      const command = pluginModule[commandName]
      commands.push({ ...command, source: 'sdk' })
    }
  }
  
  return commands
}
```

### 4.4 `/plugin` 命令管理

```typescript
// /plugin install <package> — 安装插件
// /plugin list — 列出已安装插件
// /plugin enable <name> — 启用插件
// /plugin disable <name> — 禁用插件
// /plugin remove <name> — 卸载插件

async function installPlugin(packageName: string): Promise<void> {
  // 1. npm/bun install 到插件目录
  await exec(`bun add ${packageName}`, { cwd: PLUGINS_DIR })
  
  // 2. 验证插件格式
  const pkg = await loadPluginPackage(join(PLUGINS_DIR, 'node_modules', packageName))
  if (!validatePlugin(pkg)) {
    throw new Error(`插件 ${packageName} 格式无效`)
  }
  
  // 3. 更新插件注册表
  await registerPlugin(packageName, pkg)
  
  // 4. 清除插件命令缓存
  clearPluginCommandCache()
  clearPluginSkillsCache()
}
```

## 5. Hooks 系统（`commands/hooks/`）

用户可以定义生命周期 Hooks，在特定事件时触发脚本：

### 5.1 Hook 类型

```typescript
type HookType =
  | 'PreToolUse'     // 工具执行前
  | 'PostToolUse'    // 工具执行后
  | 'Stop'           // 对话结束时
  | 'UserPromptSubmit' // 用户提交消息时
  | 'SessionStart'   // 会话开始时
  | 'Notification'   // 收到通知时
```

### 5.2 Hook 配置（在 settings.json 中）

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "BashTool",
        "command": "sh ~/.claude/hooks/pre-bash.sh"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "FileEditTool",
        "command": "npx prettier --write $CLAUDE_FILE_PATH"
      }
    ],
    "Stop": [
      {
        "command": "notify-send 'Claude 完成了任务'"
      }
    ]
  }
}
```

### 5.3 Hook 执行（`query/stopHooks.ts`）

```typescript
// 执行 stop hook（对话结束时触发）
async function handleStopHooks(
  messages: Message[],
  context: ToolUseContext,
): Promise<HookResult> {
  const stopHooks = getHooksForType('Stop', context)
  
  for (const hook of stopHooks) {
    // 将最后一条 AI 消息作为环境变量传入
    const env = {
      CLAUDE_RESPONSE: extractLastAssistantMessage(messages),
      CLAUDE_CWD: context.cwd,
    }
    
    const result = await exec(hook.command, { env })
    
    // 如果 hook 返回非零退出码且 hook.continue 为 false，停止
    if (result.exitCode !== 0 && !hook.continue) {
      return { shouldContinue: false, reason: result.stderr }
    }
    
    // 如果 hook 有 stdout，将其作为新的用户消息注入
    if (result.stdout.trim()) {
      return {
        shouldContinue: true,
        injectedMessage: result.stdout.trim(),
      }
    }
  }
  
  return { shouldContinue: false }
}
```

## 7. 技能 vs 插件 vs 命令对比

| 特性 | 技能（Skill）| 插件（Plugin）| 内置命令（Command）|
|------|------------|------------|-----------------|
| 定义方式 | Markdown 文件 | NPM 包 | TypeScript 代码 |
| 触发方式 | Slash 命令 | Slash 命令 + 工具 | Slash 命令 |
| 可提供工具 | ❌ | ✅ | ❌ |
| 可提供 Hooks | ❌ | ✅ | ❌ |
| 发布分发 | Git/文件共享 | npm publish | PR 到 Claude Code |
| 权限控制 | allowed-tools 白名单 | 完整权限配置 | 无 |
| 用户可创建 | ✅ | ✅ | ❌（需 PR）|

## 8. 设计模式

### 8.1 约定优于配置（Convention over Configuration）
技能只需放在特定目录下，无需注册，Claude Code 自动发现。

### 8.2 渐进式扩展
从简单到复杂：Slash 命令 → 技能文件 → 插件包，用户可以根据需求选择适合的扩展方式。

### 8.3 沙箱化（allowed-tools）
技能可以限制自己可以使用的工具，防止滥用权限（如只读技能只允许 FileReadTool、GrepTool）。
