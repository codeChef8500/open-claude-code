## 1. 模块职责概述

会话存储系统负责：
- **实时录制**：将每轮对话消息持久化到磁盘（`.jsonl` 文件）
- **会话恢复**：支持 `/resume` 命令加载历史会话
- **历史查询**：支持历史搜索和会话列表展示
- **多进程安全**：写队列机制确保并发安全

## 4. `utils/sessionStorage.ts` — 写入核心

### 4.1 写队列（防止并发写入冲突）

```typescript
// 使用异步写入队列（FIFO），每次只有一个写操作执行
class WriteQueue {
  private queue: Array<() => Promise<void>> = []
  private isProcessing = false
  
  async enqueue(operation: () => Promise<void>): Promise<void> {
    return new Promise((resolve, reject) => {
      this.queue.push(async () => {
        try {
          await operation()
          resolve()
        } catch (err) {
          reject(err)
        }
      })
      
      if (!this.isProcessing) {
        this.processNext()
      }
    })
  }
  
  private async processNext(): Promise<void> {
    if (this.queue.length === 0) {
      this.isProcessing = false
      return
    }
    
    this.isProcessing = true
    const next = this.queue.shift()!
    await next()
    await this.processNext()
  }
}
```

### 4.2 懒 JSON 序列化

```typescript
// 为了最小化阻塞时间，使用 100ms 延迟批量序列化
// 避免每次小更新都立即序列化整个消息对象

const LAZY_STRINGIFY_DELAY_MS = 100

class LazyJsonStringify {
  private pendingFlush: NodeJS.Timeout | null = null
  private cache: string | null = null
  
  schedule(data: unknown, onFlush: (json: string) => void): void {
    this.cache = null  // 标记为脏
    
    if (this.pendingFlush) clearTimeout(this.pendingFlush)
    
    this.pendingFlush = setTimeout(() => {
      this.cache = JSON.stringify(data)
      onFlush(this.cache)
    }, LAZY_STRINGIFY_DELAY_MS)
  }
}
```

### 4.3 `recordTranscript()` — 主录制函数

```typescript
// 被 QueryEngine 在每次消息变化时调用
export async function recordTranscript(messages: Message[]): Promise<void> {
  const sessionId = getSessionId()
  const filePath = getSessionFilePath(sessionId)
  
  // 使用写队列确保串行写入
  await writeQueue.enqueue(async () => {
    // 计算需要追加的新消息（差量更新，避免全量重写）
    const existingCount = await getExistingMessageCount(filePath)
    const newMessages = messages.slice(existingCount)
    
    if (newMessages.length === 0) return
    
    // 追加新行到 JSONL 文件
    const lines = newMessages.map(msg => JSON.stringify(msg) + '\n')
    await appendFile(filePath, lines.join(''))
    
    // 更新元数据索引
    await updateSessionMetadata(sessionId, {
      lastMessageAt: Date.now(),
      messageCount: messages.length,
      title: extractTitle(messages),
    })
  })
}
```

### 6.1 Compact Boundary 处理

会话文件中可能包含 compact_boundary 标记，恢复时需要正确处理：

```typescript
function getMessagesAfterCompactBoundary(messages: Message[]): Message[] {
  // 从后往前找最近的 compact_boundary
  const boundaryIdx = messages.findLastIndex(
    m => m.type === 'system' && m.subtype === 'compact_boundary'
  )
  
  if (boundaryIdx === -1) {
    return messages  // 没有压缩边界，返回全部
  }
  
  // compact_boundary 之后的消息（包含边界本身，它包含摘要）
  return messages.slice(boundaryIdx)
}
```

```typescript
// 存储在 ~/.claude/history（无扩展名）

const MAX_HISTORY_ENTRIES = 1000

export async function saveToHistory(prompt: string): Promise<void> {
  // 避免保存空提示词和重复提示词
  if (!prompt.trim()) return
  
  const history = await loadHistory()
  
  if (history[history.length - 1] === prompt) return  // 与上次相同
  
  const updated = [...history.slice(-MAX_HISTORY_ENTRIES + 1), prompt]
  await writeFile(HISTORY_PATH, updated.join('\n'))
}

// 历史搜索（Ctrl+R）
export async function searchHistory(query: string): Promise<string[]> {
  const history = await loadHistory()
  return history
    .filter(h => h.toLowerCase().includes(query.toLowerCase()))
    .reverse()  // 最近的在前
    .slice(0, 20)  // 最多显示 20 条
}
```

## 11. 设计模式

### 11.1 增量写入（Append-Only）
JSONL 格式允许增量追加，不需要重写整个文件，对长会话非常高效。

### 11.2 写队列（Sequential Writes）
异步写队列保证：即使多个 React 渲染周期在短时间内触发 `recordTranscript`，文件写入也是串行、有序的。

### 11.3 Compact Boundary 分层
compact_boundary 消息作为会话文件中的"检查点"，将压缩前和压缩后的历史分隔，恢复时只需加载最新检查点之后的内容，节省内存。

### 11.4 元数据与数据分离
会话数据（JSONL）和元数据（JSON 索引）分开存储，列表展示时只需加载轻量的元数据，不需要读取完整对话内容。
