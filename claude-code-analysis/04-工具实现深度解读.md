### 2.2 权限系统（`bashPermissions.ts`）

BashTool 拥有最复杂的权限系统，支持通配符匹配：

```typescript
// 权限规则示例：
// "git commit*"  → 允许所有 git commit 命令
// "npm run *"    → 允许所有 npm run 命令
// "rm -rf *"     → 明确拒绝

async function bashToolHasPermission(
  command: string,
  permissionContext: ToolPermissionContext,
): Promise<PermissionResult> {
  // 1. 解析命令的所有 shell 操作符（&&, ||, ;, |）
  const parts = splitCommandWithOperators(command)
  
  // 2. 对每个子命令检查权限规则
  for (const part of parts) {
    const result = checkSingleCommand(part, permissionContext)
    if (result.behavior !== 'allow') return result
  }
  
  return { behavior: 'allow' }
}
```

**`preparePermissionMatcher()`** 方法实现了 BashTool 特有的通配符匹配：

```typescript
async preparePermissionMatcher(input) {
  return async (pattern: string) => {
    // 支持：git commit*、npm run build、cd *

  }
}
```

## 10. 工具执行编排（`services/tools/toolOrchestration.ts`）

这是实际调用工具的核心模块：

### 10.1 `runTools()` — 工具并发执行

```typescript
async function runTools(
  toolUseBlocks: ToolUseBlock[],
  canUseTool: CanUseToolFn,
  toolUseContext: ToolUseContext,
): Promise<(UserMessage | AttachmentMessage)[]> {
  
  // 1. 将工具分组：并发安全的工具可以同时运行
  const concurrentGroups = groupByConcurrency(toolUseBlocks)
  
  const results: (UserMessage | AttachmentMessage)[] = []
  
  for (const group of concurrentGroups) {
    // 同一 group 内的工具并行执行
    const groupResults = await Promise.all(
      group.map(toolUse => executeSingleTool(toolUse, canUseTool, toolUseContext))
    )
    results.push(...groupResults.flat())
  }
  
  return results
}
```

### 10.2 并发安全分组

```typescript
// isConcurrencySafe 决定是否可以与其他工具并行
function groupByConcurrency(toolUseBlocks: ToolUseBlock[]) {
  const groups: ToolUseBlock[][] = []
  let currentGroup: ToolUseBlock[] = []
  
  for (const tool of toolUseBlocks) {
    const toolDef = findToolByName(tools, tool.name)
    if (toolDef?.isConcurrencySafe(tool.input)) {
      currentGroup.push(tool)  // 加入当前并发组
    } else {
      if (currentGroup.length > 0) groups.push(currentGroup)
      currentGroup = [tool]  // 非并发安全：单独一组
      groups.push(currentGroup)
      currentGroup = []
    }
  }
  
  return groups
}
```

### 10.3 单工具执行流程

```typescript
async function executeSingleTool(toolUse, canUseTool, context) {
  const tool = findToolByName(context.tools, toolUse.name)
  
  // 1. 输入验证
  const parseResult = tool.inputSchema.safeParse(toolUse.input)
  if (!parseResult.success) return createErrorResult(parseResult.error)
  
  // 2. 额外语义验证
  if (tool.validateInput) {
    const validation = await tool.validateInput(parseResult.data, context)
    if (!validation.result) return createErrorResult(validation.message)
  }
  
  // 3. 权限检查
  const permission = await canUseTool(tool, parseResult.data, context)
  if (permission.behavior === 'deny') return createDeniedResult(permission.message)
  
  // 4. 执行工具
  const result = await tool.call(parseResult.data, context, canUseTool, parentMessage, onProgress)
  
  // 5. 处理 contextModifier（如果工具需要修改上下文）
  if (result.contextModifier) {
    // 注意：只有非并发安全的工具才能使用 contextModifier
    updatedContext = result.contextModifier(context)
  }
  
  return createToolResult(toolUse.id, result.data)
}
```

## 11. 工具设计模式总结

### 11.1 文件操作工具的防护层次

```
用户输入 → validateInput（语义）→ checkPermissions（权限）→ call（执行）
    ↓              ↓                      ↓                   ↓
 格式检查      路径遍历检查           用户确认弹框          实际 I/O
 类型检查      写保护目录检查         allow/deny/ask        原子写入
 范围检查      MCP 权限匹配          timestamp 校验         事务回滚
```

### 11.2 大结果磁盘溢出策略

所有工具都通过 `maxResultSizeChars` 控制内存使用：

| 工具 | maxResultSizeChars |
|------|-------------------|
| FileReadTool | `Infinity`（自有限制机制）|
| BashTool | `50_000` |
| GrepTool | `100_000` |
| FileEditTool | `100_000` |
| AgentTool | `Infinity`（Agent 结果不限制）|
| WebFetchTool | `500_000` |

### 11.3 UI 折叠分类（`isSearchOrReadCommand`）

工具通过返回 `{ isSearch, isRead, isList }` 告知 UI 是否折叠显示：
- `isSearch: true` → 显示 "Searched N files"
- `isRead: true` → 显示 "Read N files"
- `isList: true` → 显示 "Listed N directories"
