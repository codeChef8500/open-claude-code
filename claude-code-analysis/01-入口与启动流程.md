# Claude Code 源码深度解读 — 01 入口与启动流程

> 覆盖文件：`entrypoints/cli.tsx`、`main.tsx`、`entrypoints/init.ts`、`bootstrap/state.ts`

## 1. 模块职责概述

启动层的职责是：**以最快速度启动 CLI 并路由到正确的执行模式**。

整个启动链分为三个关键文件：

```
entrypoints/cli.tsx          ← 真正的进程入口（Bun 执行）
    └── main.tsx             ← 完整 CLI（Commander.js + React/Ink）
         └── entrypoints/init.ts  ← 基础设施初始化（遥测、网络、配置）
              └── bootstrap/state.ts  ← 全局单例状态（进程生命周期内存活）
```

## 3. 启动流程详解

### 3.1 阶段一：进程入口 (`entrypoints/cli.tsx`)

Bun 执行 `cli.tsx`，这是**全程最轻量的入口**，所有导入都是动态的，目的是最小化模块加载：

```typescript
async function main(): Promise<void> {
  const args = process.argv.slice(2);

  // ① 快速路径：--version (0 模块加载)
  if (args[0] === '--version') {
    console.log(`${MACRO.VERSION} (Claude Code)`);
    return;
  }
  
  // ② 快速路径：bridge 模式（IDE 集成）
  if (feature('BRIDGE_MODE') && (args[0] === 'remote-control' || ...)) {
    const { bridgeMain } = await import('../bridge/bridgeMain.js');
    await bridgeMain(args.slice(1));
    return;
  }
  
  // ③ 快速路径：daemon 模式
  // ④ 快速路径：后台会话 (ps/logs/attach/kill)
  // ⑤ 快速路径：MCP、Chrome 扩展、Computer Use
  // ... 更多快速路径 ...
  
  // ⑥ 通用路径：加载完整 CLI
  const { startCapturingEarlyInput } = await import('../utils/earlyInput.js');
  startCapturingEarlyInput();  // 在 main 加载期间缓存用户提前输入的字符
  
  const { main: cliMain } = await import('../main.js');  // 懒加载主模块
  await cliMain();
}
```

**关键设计**：快速路径机制确保简单命令（如 `--version`）不加载 React/Ink/Commander 等重型依赖。

#### 已知的快速路径列表

| 条件 | 目标模块 |
|------|---------|
| `--version` / `-v` | 直接 `console.log`，零导入 |
| `--dump-system-prompt` | `constants/prompts.js`（ant-only） |
| `--claude-in-chrome-mcp` | `utils/claudeInChrome/mcpServer.js` |
| `--chrome-native-host` | `utils/claudeInChrome/chromeNativeHost.js` |
| `remote-control` / `bridge` | `bridge/bridgeMain.js` |
| `daemon` | `daemon/main.js` |
| `ps`/`logs`/`attach`/`kill`/`--bg` | `cli/bg.js` |
| `new`/`list`/`reply` (TEMPLATES) | `cli/handlers/templateJobs.js` |
| `environment-runner` | `environment-runner/main.js` |
| `self-hosted-runner` | `self-hosted-runner/main.js` |
| `--worktree --tmux` | `utils/worktree.js` |

### 3.2 阶段二：并行预加载 (`main.tsx` 顶部副作用)

`main.tsx` 是 Claude Code 最大的单个文件（803KB）。文件顶部的**副作用导入**在所有其他模块加载的同时并行执行：

```typescript
// ① 性能打点（模块加载前）
import { profileCheckpoint } from './utils/startupProfiler.js';
profileCheckpoint('main_tsx_entry');   // 记录进入 main.tsx 的时间

// ② MDM 设置读取（macOS plutil / Windows reg query）
import { startMdmRawRead } from './utils/settings/mdm/rawRead.js';
startMdmRawRead();   // 启动子进程，并行读取企业 MDM 策略

// ③ Keychain 预读（OAuth token + API key）
import { startKeychainPrefetch } from './utils/secureStorage/keychainPrefetch.js';
startKeychainPrefetch();  // 预读 macOS Keychain，避免后续串行等待（~65ms）

// ... 其他 135+ 个模块导入在此期间并行加载 ...

profileCheckpoint('main_tsx_imports_loaded');
```

这三个副作用在约 **135ms** 的模块评估期间并行运行，消除了串行等待带来的启动延迟。

### 3.3 阶段三：基础设施初始化 (`entrypoints/init.ts`)

`init()` 函数通过 `lodash.memoize` 保证只执行一次，完成：

```typescript
export const init = memoize(async (): Promise<void> => {
  // 1. 启用配置系统（解析 settings.json, CLAUDE.md 等）
  enableConfigs()
  
  // 2. 应用"安全"环境变量（信任对话框之前）
  applySafeConfigEnvironmentVariables()
  
  // 3. 注入 CA 证书（TLS 连接前，Bun 的 BoringSSL 只在启动时加载一次）
  applyExtraCACertsFromConfig()
  
  // 4. 注册优雅关闭处理器
  setupGracefulShutdown()
  
  // 5. 并行启动非阻塞初始化任务（fire-and-forget）
  void Promise.all([
    import('./services/analytics/firstPartyEventLogger.js'),
    import('./services/analytics/growthbook.js'),
  ]).then(([fp, gb]) => {
    fp.initialize1PEventLogging()
    gb.onGrowthBookRefresh(...)
  })
  
  // 6. 预连接 Anthropic API（减少首次请求 TCP 握手延迟）
  void preconnectAnthropicApi()
  
  // 7. 初始化远程管理设置加载 Promise（允许其他系统 await）
  if (isEligibleForRemoteManagedSettings()) {
    initializeRemoteManagedSettingsLoadingPromise()
  }
  
  // 8. 配置全局 mTLS + HTTP 代理
  configureGlobalMTLS()
  configureGlobalAgents()
  
  // 9. 懒加载遥测（OpenTelemetry ~400KB，仅在需要时加载）
  // 10. 初始化 LSP 服务器管理器
  // 11. Windows 特殊处理（shell 路径）
})
```

**关键点**：遥测（OpenTelemetry + gRPC，合计 ~1.1MB）通过动态 `import()` 在真正需要时才加载，不阻塞启动。

### 3.4 阶段四：Commander.js 命令树 (`main.tsx` → `cliMain()`)

`main.tsx` 中的 `cliMain()` 函数构建完整的 Commander.js 命令树。主命令注册核心选项：

  component: React.ReactElement,

## 7. 设计模式

### 7.1 快速路径（Fast Path）模式
通过提前检查 `process.argv` 分发到轻量级处理器，避免加载完整 CLI 的 800KB 代码。

### 7.2 副作用并行化（Side-Effect Parallelization）
将耗时的 I/O 操作（MDM 读取、Keychain 读取）作为**模块级副作用**，在 TypeScript 模块加载期间并行执行。

### 7.3 懒加载（Lazy Loading）
大型模块（OpenTelemetry、gRPC、React/Ink）通过动态 `import()` 延迟加载，仅在确实需要时加载。

### 7.4 单例初始化（Singleton Init）
`init()` 函数通过 `memoize` 保证在进程生命周期内只执行一次，无论被调用多少次。

### 7.5 死代码消除（Dead Code Elimination）
编译时通过 `feature('FLAG_NAME')` 调用，Bun bundle 将未激活的分支完全删除，外部构建产物更小更安全。

## 10. 设计亮点与潜在问题

### 亮点
- **分层快速路径**：大部分命令走的都是轻量路径，TTFR 极短
- **并行预加载**：把 I/O 等待时间折叠进模块加载时间，几乎零额外成本
- **memoize init**：防止多处调用造成重复初始化
- **早期输入捕获**：消除了模块加载期间的输入延迟感知

### 潜在问题
- `main.tsx` 文件高达 803KB，职责过于集中，难以维护
- `bootstrap/state.ts` 虽然有"不要随意添加"的注释，但随着功能增加，状态字段已非常多（~50 个字段）
- `init()` 的 `memoize` 在测试场景中需要额外处理（测试间状态污染）
